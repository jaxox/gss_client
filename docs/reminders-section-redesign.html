üèÜ Recommended: Option 1 (Always-Visible Toggle Cards)

This provides the best user experience for Step 2 because:

  • Discoverability: Users immediately see both reminder options without hunting
  • Speed: One tap to enable with smart defaults (7d RSVP, 24h Event)
  • Consistency: Similar to modern mobile settings patterns (iOS, Android)
  • Progressive disclosure: Simple toggle first, tap card for advanced timing
  • Status at-a-glance: Colored highlight shows what's enabled
  • No modal required: For basic enable/disable (60-70% less friction)

Implementation Notes:

  • Tap toggle button: Enable/disable with defaults
  • Tap card body: Opens modal for custom timing
  • Visual feedback: Enabled cards have orange tint + border
  • Defaults: 7 days for RSVP, 24 hours for Event
  • Space cost: ~120px vertical (vs ~50px for header+FAB)

Why Not Other Options:

  • Option 2: Too subtle, users might not realize it's interactive
  • Option 3: Empty state wastes space, still requires modal for everything
  • Option 4: Clever but complex state changes confuse users
